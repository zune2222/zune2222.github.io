"내가 개발자인데, 내 블로그는 내가 만들어야지"

그때부터 시작된 프로젝트가 지금의 zun2log가 되었다.

## 기획 단계: 남들과 다른 걸 만들고 싶었다

근데 배경을 그리다가 생각이 바뀌었다. "그냥 그림보다... 살아있는 배경이면 어떨까?"

## 기술적 도전과 해결

### 1. Canvas 파티클 최적화
처음엔 파티클 1000개만 렌더링해도 프레임이 떨어졌다.

```javascript
// 문제: 매 프레임마다 모든 파티클 업데이트
particles.forEach(particle => {
  particle.update();
  particle.draw(ctx);
});

// 해결: 화면 밖 파티클 제거
particles = particles.filter(particle => 
  particle.opacity > 0 && particle.y < canvas.height + 100
);
```

### 2. 날씨별 파티클 구현
각 날씨마다 다른 느낌을 주고 싶었다:
- **비**: 빗줄기가 땅에 닿으면 물방울로 변하는 효과
- **눈**: 천천히 흔들리며 내리는 움직임
- **구름**: 여러 원이 모여 입체적인 구름 형성
- **태양**: 시간대별로 색이 변하는 광선
- **별**: 반짝이는 효과와 마우스 인터랙션

### 3. 모듈화 작업
코드가 600줄을 넘어가면서 유지보수가 힘들어졌다. 그래서 모듈화를 진행했다:
```
homePic/
├── Particle.js       // 파티클 클래스
├── weatherConfig.js  // 날씨 설정
├── weatherAPI.js     // API 통신
└── canvasUtils.js    // 캔버스 유틸
```
