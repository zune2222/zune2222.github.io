## 시작: 개발자로서의 나만의 공간

개발자라면 누구나 한번쯤 자신만의 블로그를 만들어보고 싶다는 생각을 하지 않을까? 나 역시 마찬가지였다. 
하지만 단순히 글을 올리는 블로그가 아닌, **나의 개성과 기술력을 동시에 보여줄 수 있는 공간**을 만들고 싶었다.

그래서 결정했다. 템플릿을 쓰지 않고, 처음부터 끝까지 내가 직접 만들겠다고.

## 핵심 컨셉: 여행하는 네비게이터

블로그의 가장 큰 특징은 독특한 네비게이션 시스템이다. 

### Travel Button - 블로그를 여행하다

이름을 'Travel Button'으로 지은 이유는 단순하다. **내 블로그를 여행하는 느낌**을 주고 싶었기 때문이다.
각 페이지는 하나의 여행지처럼 다른 테마와 분위기를 가지고 있고, 방문자들이 버튼을 클릭해 이동할 때마다 
새로운 곳으로 여행을 떠나는 듯한 경험을 하길 바랐다.

```javascript
// 클릭하면 펼쳐지는 네비게이션
const springs = useSpring(
  currentMode === false
    ? {
        width: "4.3rem",
        height: "4.3rem",
      }
    : {
        width: "20rem",
        height: "8rem",
      }
);
```

일반적인 네비게이션 바 대신, 화면 상단에 떠있는 작은 버튼을 클릭하면 
부드러운 애니메이션과 함께 3개의 목적지가 나타난다:

- **Home** - 메인 페이지
- **Log** - 개발 일지  
- **Memory** - 추억 기록

각 페이지마다 다른 아이콘이 표시되고, 현재 위치에 따라 버튼의 이미지가 변경된다.
`@react-spring/web`을 사용해 자연스러운 트랜지션 효과를 구현했다.

## 아이패드로 그린 배경 일러스트

블로그의 모든 배경 이미지는 직접 아이패드로 그렸다. 

### 메인 페이지: 겨울 풍경
메인 페이지의 배경은 눈 내리는 겨울 풍경이다. 하얀 눈과 푸른 하늘, 그리고 작은 집들이 어우러진 평화로운 장면을 그렸다.

### Log 페이지: 책과 데이지
개발 일지를 기록하는 Log 페이지는 책과 데이지 꽃을 테마로 했다. 지식을 쌓아가는 과정을 책으로, 
새로운 시작과 순수함을 데이지로 표현했다.

### Memory 페이지: 파리의 추억
Memory 페이지는 파리를 배경으로 한다. 사실 아직 파리에 가본 적은 없지만, **여행의 낭만**하면 떠오르는 곳이 파리였다.
에펠탑과 파리의 거리를 그리면서 '언젠가는 꼭 가보리라'는 다짐을 담았다. 
추억을 기록하는 공간에 미래의 여행지를 그려넣은 것도 나름의 의미가 있다고 생각한다.

## 인터랙티브한 요소들

### 1. 날씨 기반 파티클 시스템

가장 공을 들인 부분 중 하나는 메인 페이지의 날씨 시스템이다.

```javascript
// 날씨 API로 실시간 날씨 정보 가져오기
const weatherData = await fetchWeatherData();

// 날씨에 따른 파티클 효과
switch (weatherType) {
  case "cloud":
    // 구름 효과
    break;
  case "sun":
    // 태양광선 효과
    break;
  case "rain":
    // 비 효과
    break;
  case "star":
    // 별 효과 (밤)
    break;
}
```

사용자의 위치(부산)를 기반으로 실제 날씨를 받아와서:
- **맑은 날**: 태양광선이 내리쬐는 효과
- **흐린 날**: 구름이 떠다니는 효과
- **비오는 날**: 빗방울이 떨어지는 효과
- **밤**: 반짝이는 별들

각각의 파티클은 마우스와 상호작용한다. 마우스를 가까이 가져가면 구름이 밀려나고, 
별은 더 밝게 빛나며, 빗줄기는 휘어진다.

### 2. 마우스 호버 효과

```javascript
// 3D 카드 호버 효과
const handleMouseMove = (e) => {
  const rect = e.currentTarget.getBoundingClientRect();
  const x = ((e.clientX - rect.left) / rect.width - 0.5) * 20;
  const y = ((e.clientY - rect.top) / rect.height - 0.5) * -20;
  
  setTransform(`rotateY(${x}deg) rotateX(${y}deg)`);
};
```

프로필 이미지에 마우스를 올리면 3D 효과로 기울어지며 반응한다. 
작은 디테일이지만 사용자 경험을 한층 더 풍부하게 만들어준다.

## 개발 과정

### 2023년 5월: 프로젝트 시작
Next.js 13과 Tailwind CSS를 기반으로 프로젝트를 시작했다. 
App Router를 사용해 파일 기반 라우팅을 구현하고, MDX로 콘텐츠를 관리하기로 결정했다.

### 2023년 9월: 기본 구조 완성
메인 페이지, Log, Memory 페이지의 기본 구조를 완성했다. 
각 페이지별 레이아웃과 네비게이션 시스템을 구현했다.

### 2023년 10월: 인터랙티브 요소 추가
- Travel Button 애니메이션 구현
- 3D 카드 호버 효과 추가
- 페이지 전환 애니메이션 적용

### 2024년 11월: 성능 최적화
- 이미지 로딩 최적화 (Next.js Image 컴포넌트 활용)
- 폰트 최적화 (로컬 폰트 사용)
- 빌드 시스템 개선

### 2025년 1월: 날씨 시스템 구현
가장 큰 업데이트였다. Canvas API를 활용한 파티클 시스템을 구현하고, 
실시간 날씨 데이터와 연동했다.

```javascript
// 파티클 시스템 모듈화
├── Particle.js       // 파티클 클래스
├── weatherAPI.js     // 날씨 API 연동
├── weatherConfig.js  // 날씨별 설정
├── canvasUtils.js    // 캔버스 유틸리티
└── index.js          // 메인 컴포넌트
```

### 2025년 5월: 모듈화 및 최적화
코드를 깔끔하게 정리하고 모듈화했다. 각 기능을 독립적인 파일로 분리해 
유지보수성을 높이고, 성능을 최적화했다.

## 기술 스택

- **Framework**: Next.js 13 (App Router)
- **Styling**: Tailwind CSS
- **Animation**: React Spring, Canvas API
- **Content**: MDX, Contentlayer
- **Deployment**: GitHub Pages
- **기타**: 
  - TypeScript
  - Google Analytics
  - Giscus (댓글 시스템)

## 배운 점과 느낀 점

### 1. 디테일의 중요성
작은 애니메이션 하나, 색상 하나가 전체적인 사용자 경험을 크게 좌우한다는 것을 깨달았다. 
특히 파티클 시스템을 구현하면서 수많은 미세 조정을 거쳤는데, 그 과정에서 디테일의 힘을 실감했다.

### 2. 성능과 미적 요소의 균형
아름다운 효과를 추가하면서도 성능을 해치지 않는 것이 중요했다. 
Canvas 최적화, 애니메이션 프레임 관리, 이미지 로딩 최적화 등 다양한 기법을 활용했다.

### 3. 개인 프로젝트의 즐거움
회사 프로젝트와 달리 내가 원하는 대로, 내 속도대로 만들 수 있다는 것이 큰 즐거움이었다. 
새벽에 문득 아이디어가 떠오르면 바로 구현해볼 수 있는 자유로움이 좋았다.

## 앞으로의 계획

- **다크 모드 지원**: 시간대에 따른 자동 테마 전환
- **더 많은 날씨 효과**: 눈, 안개, 번개 등
- **모바일 최적화**: 터치 인터랙션 개선
- **콘텐츠 확장**: 더 많은 개발 경험과 이야기 공유

## 마무리

개발자에게 개인 블로그는 단순한 글쓰기 공간 이상의 의미를 가진다. 
자신의 기술을 실험하고, 창의성을 발휘하며, 성장을 기록하는 공간이다.

내 블로그가 방문자들에게 단순히 정보를 전달하는 것을 넘어, 
**작은 여행**을 선물하는 공간이 되기를 바란다. Travel Button을 클릭할 때마다 
새로운 세계로 이동하는 듯한 경험, 각 페이지마다 다른 분위기를 느끼는 즐거움을 주고 싶다.

무엇보다 **"개발자로서의 나는 이런 사람이야"**라고 보여줄 수 있는, 
나만의 디지털 명함이 된 것 같아 뿌듯하다.
